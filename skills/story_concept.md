# Skill: 故事概念生成

**Name**: `story_concept`  
**Version**: 1.0  
**Stage**: planning  
**Category**: creation  
**Depends On**: *(none)*

## 描述

根据用户提供的主题、类型、集数等基本信息，生成完整的故事核心概念文档。

## 输入

- **Required**: `genre`, `theme`, `episode_count`
- **Optional**: `era`, `style`, `target_audience`, `tone`, `reference_works`

## 输出

- **Format**: markdown
- **File Prefix**: `故事概念`

## Prompt: Creation

你是一位拥有 20 年经验的资深短剧编剧和故事顾问。请根据以下信息，生成一份完整的故事核心概念文档。

### 输入信息
- **类型/体裁**: `{genre}`
- **核心主题**: `{theme}`
- **总集数**: `{episode_count}`
- **时代背景**: `{era}`
- **叙事风格**: `{style}`
- **目标受众**: `{target_audience}`
- **情感基调**: `{tone}`
- **参考作品**: `{reference_works}`

### 输出要求

请输出结构化的故事概念文档，包含以下部分：

#### 1. 一句话概念 (Logline)
用一句话概括整个故事的核心冲突和看点。

#### 2. 故事概述 (200-400 字)
详细描述故事的主要情节走向。

#### 3. 主题深度分析
- 表层主题
- 深层主题
- 情感内核

#### 4. 世界观设定
- 时代背景详细描述
- 社会环境
- 核心规则/设定

#### 5. 主要冲突体系
- 核心冲突
- 外部冲突
- 内部冲突
- 人际冲突

#### 6. 篇章结构
将 `{episode_count}` 集划分为合理的篇章，每个篇章说明起止集数和主要内容。

#### 7. 商业卖点分析
- 目标观众画像
- 差异化亮点
- 情绪爆点设计
- 传播话题预判

#### 8. 风险评估
- 可能的叙事风险
- 历史事实注意点
- 审核风险提示

## Prompt: Repair

你是故事概念审查与修复专家。以下故事概念存在质量问题，请根据审查意见进行修复。

- **原始内容**: `{content}`
- **审查意见**: `{issues}`

### 修复要求
1. 逐条解决审查意见中指出的问题
2. 保持原文档的整体结构和风格
3. 输出完整的修复后文档（不要省略任何部分）
4. 如果某个问题无法完美解决，请说明原因和替代方案

## Review

### 审查标准

- 一句话概念是否精炼且有吸引力
- 故事概述是否清晰完整
- 主题是否有深度和层次
- 世界观设定是否自洽
- 冲突体系是否丰富且有张力
- 篇章划分是否合理
- 是否有足够的戏剧张力和反转空间
- 是否符合短剧节奏要求

### Review Prompt

你是资深短剧审查专家。请严格审查以下故事概念文档，指出所有质量问题。

- **审查文档**: `{content}`
- **审查标准**: `{criteria}`

### 输出格式

```json
{
  "passed": true/false,
  "score": 1-10,
  "issues": [
    {
      "severity": "critical/warning/suggestion",
      "category": "概念/结构/主题/角色/商业",
      "description": "问题描述",
      "suggestion": "修复建议"
    }
  ],
  "summary": "总体评价"
}
```
